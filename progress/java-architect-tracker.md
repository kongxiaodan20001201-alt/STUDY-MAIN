### C. 分布式系统架构 (新增重点)

- **[NEW] C.15 - 分布式事务深度解析**（2026-01-10）
  - **信心水平**：精通
  - **掌握内容**：
    - **Seata AT 模式**：理解 "一阶段提交 + Undo Log + 全局锁" 的高性能原理；掌握 XID 的生成与传递机制。
    - **脏读权衡**：理解 AT 模式默认存在物理脏读，掌握 `@GlobalLock` 或 "中间态设计" 的解决方案。
    - **架构选型**：
      - **核心资金链路** -> TCC (高性能强一致)。
      - **普通业务链路** -> Seata AT (开发效率高)。
      - **附带业务 (如送积分)** -> 本地消息表 + MQ (最终一致性，用户体验最佳)。

- **[NEW] C.16 - 分布式锁原理**（2026-01-10）
  - **信心水平**：高
  - **掌握内容**：
    - **Redis**：`SetNX + TTL` (高性能，需处理锁续期，有丢锁风险)。
    - **Zookeeper**：`临时顺序节点 + Watcher` (强一致，无死锁，适合对数据严谨性要求高的场景)。

- **C.14 - CAP 定理与 BASE 理论**（2026-01-09）
  - **关键理解**：P (分区容错) 必选；CP (Zookeeper/金融) vs AP (Redis/Eureka/互联网)。

- **C.20 - Redis 架构演进**（2026-01-09）
  - **关键理解**：Sentinel (高可用) vs Cluster (分片)；Redlock 在 Cluster 下的局限性。

## 🏭 生产案例库更新

- **[Distributed Tx]** 积分发放业务不使用 Seata AT，而是采用 **本地消息表 + MQ** 方案，确保订单提交成功后异步必达，避免回滚导致的积分忽有忽无，提升用户体验。
- **[Distributed Lock]** 避免在 Redis Cluster 模式下迷信 Redlock，高并发场景优先使用 Redis 普通锁 + 数据库乐观锁兜底。
